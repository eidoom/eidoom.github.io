Autostarting services with systemd

   02-12-2019      1 min read      computing      linux · server · systemd

Introduction

There are sufficiently many services running on my home server now that starting them all manually after I restart the machine has become tedious, as infrequently as it occurs. Systemd can be used to automate this.

Case study

One such service is beets web. To make it run on system start-up, simply create a .service file in /etc/systemd/system/

$ sudo vim /etc/systemd/system/beetsweb.service

[Unit]
Description=Beets web daemon
After=network.target

[Service]
User=ryan
Group=ryan

ExecStart=python3 -m beets web

Type=simple
TimeoutStopSec=20
KillMode=process
Restart=on-failure

[Install]
WantedBy=multi-user.target

To check that it works, manually start the daemon with

sudo systemctl start beetsweb

If the .service file needs changing, the configuration should be reloaded with

sudo systemctl start beetsweb

Once it’s ready to use, activate running on startup with

sudo systemctl enable beetsweb
